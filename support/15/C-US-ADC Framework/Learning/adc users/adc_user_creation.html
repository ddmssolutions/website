  • Session ID - the unique user name for use on the external device
  • User ID - The purpose of this User ID is to map the ADC User to an Business Central login for two purposes:

    • When any posting occurs via the ADC Conversations (i.e. posting a picking note) the system will flag the creatied entries and registers as this User ID

    When filtering locations and warehouse employees this user will be used

    Note #1: this does NOT have to map one-to-one or to the same user as the ADC user, additionally the user does not that user have to be enabled, nor have any permissions assigned

    i.e. A single Business Central login can be created for the warehouse employees "WHSE SCANNER" which is disabled as to not allow logon to the Business Central modern client. That user will be configured in warehouse employees as to give access to the various locations. All ADC Users are then configured with WHSE SCANNER as their User ID, as such all ledger entries, registers, posted documents will have the "created by" user as WHSE SCANNER. If more tracability is required as to which user posted which document than additional Business Central users should be created and mapped to their appropriate ADC Logins.

    Note #2: When posting the permissions of this mapped user are NOT used - the OData User (configured when connecting the ADC Modern ADC Client) requires all of the permissions to post / register
