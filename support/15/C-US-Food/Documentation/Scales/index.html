Scales



Code

Unique identifier


Description

Text to give a descriptive name to the scale record


Unit of Measure

The Unit of Measure that this scale is configure to weigh in (i.e. LBs, KG, Grams...)


Scale Codeunit ID

This identifies a codeunit object that has been designed to connect to the physical scale and return the weight upon that scale. This codeunit should have a TableNo property of "Scale.COL.US" and contain and OnRun trigger. Requests for the weight of a scale will run this codeunit (passing the scale record) and expect the "Weight.COL.US" column to be populated with the scales weight reading


Type

Select from Serial Port, IP Address, API, the selection here determines the remainder of the fields that are visible and denotes the connection method to the scale (note the Scale Codeunit ID is responsible for the actual connection). The recommend connection method for ensuring future compatability is API


Port Name

The port number that maps to or represents the physical port (Type = Serial Port)


Baud Rate

The data rate in bits per second (Type = Serial Port)


Parity

Parity is a method of detecting errors in transmission (Type = Serial Port)


Data Bits

The number of data bits in each character (Type = Serial Port)


Stop Bits

Stop bits sent at the end of every character (Type = Serial Port)


API URL

The http (or https) URL of the Rest API url service that returns the weight from the scale (Type = API)


API Key

This is the username and password (base 64 encoded) used to connect to the RestAPI url (Type = API)


Server IP Address

The IP address to be connected to (Type = IP Address)


Server Port

The port number at the IP address (Type = IP Address)


Start Char

The numeric char value that proceeds a weight reading (i.e. STX (Start of Header char) = 1)


End Char

The numeric char value that terminates a weight reading (i.e. ETX (End of Header char) = 3)


Data Retrieval Method

Two options are availble Request Data - Business Central has to request the weight on demand (i.e. when a user presses and action button "Get Weight"). Data us Pushed - the weight is continually broadcast or broadcast by interaction outside of Business Central that triggers the data to be sent. Business Central will be configured to watch for that broadcasted data and perform an action when it is received (i.e. An external button (physical button connected to the scale) is pressed, which in turn sends the data out - this is then caught by Business Central and then processed)
